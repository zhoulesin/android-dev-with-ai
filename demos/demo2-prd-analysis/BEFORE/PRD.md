# 产品需求文档：BestPractice

> 版本：v1.0 | 创建：2025-05-18 | 负责人：Tech Lead

---

## 1. 产品概述

BestPractice 是一款面向 Android 开发者的技术文章阅读 App。用户可浏览最新技术文章、搜索感兴趣的内容、收藏优质文章以便回顾。

**核心价值**：让开发者在碎片时间高效获取 Android 最佳实践知识。

**目标用户**：Android 开发者（初级到高级），已熟悉 Compose/Hilt/Room 等技术栈概念。

---

## 2. 用户故事

### 2.1 文章浏览（P0）

| ID | 用户故事 | 验收标准 |
|----|---------|---------|
| US-01 | 作为开发者，我希望能看到最新文章列表 | 按发布时间倒序；显示标题、摘要、作者、日期；加载中显示 loading |
| US-02 | 作为开发者，我希望能查看文章详情 | 点击卡片进入详情；正文支持段落阅读；有返回按钮 |
| US-03 | 作为开发者，我希望能看到加载失败提示 | 无网络/超时显示错误文案；有重试入口 |

### 2.2 收藏管理（P1）

| ID | 用户故事 | 验收标准 |
|----|---------|---------|
| US-04 | 作为开发者，我希望收藏文章 | 列表和详情页均可收藏；已收藏显示实心图标 |
| US-05 | 作为开发者，我希望查看收藏列表 | 独立页面；仅显示已收藏文章；支持取消收藏 |
| US-06 | 作为开发者，我希望收藏状态持久化 | 退出重进不丢失 |

### 2.3 搜索（P1）

| ID | 用户故事 | 验收标准 |
|----|---------|---------|
| US-07 | 作为开发者，我希望搜索文章 | 输入关键词实时搜索；匹配标题和摘要 |
| US-08 | 作为开发者，我希望看到搜索空态 | 无结果时显示友好提示 |

## 3. 功能规格

### 3.1 首页文章列表

```
┌──────────────────────────────────┐
│ TopAppBar                        │
│ BestPractice            🔍   🔖  │
├──────────────────────────────────┤
│ ArticleCard  #1             [☆]  │
│  ┌ 文章标题(max 2行) ────────┐  │
│  │ 摘要内容(max 2行)         │  │
│  │ 作者名    2025-05-18      │  │
│  └──────────────────────────┘  │
│ ArticleCard  #2             [★]  │
│  ...                            │
└──────────────────────────────────┘
```

**交互**：点击卡片 → 文章详情；点击收藏图标 → 切换收藏状态

**状态处理**：
- Loading: 居中 CircularProgressIndicator
- Error: 居中 "加载失败: xxx"
- Empty: 居中 "暂无文章"

### 3.2 文章详情

**字段**：标题(H1)、作者、发布时间、正文(Markdown)

**交互**：顶部返回按钮；页面可滚动

### 3.3 收藏列表

同上，但仅显示 isBookmarked=true 的文章。空态："还没有收藏文章"。

### 3.4 搜索

**输入框**：顶部搜索栏，输入即搜（200ms 防抖）。结果列表：标题可点击，进入详情。

## 4. 技术约束

| 约束 | 说明 |
|------|------|
| 架构 | Clean Architecture（domain / data / feature） |
| UI | Jetpack Compose + Material3 |
| DI | Hilt |
| 本地存储 | Room（缓存文章） + DataStore（偏好） |
| 导航 | Navigation Compose 单 Activity |
| 语言 | Kotlin 2.x |
| 最小 SDK | API 26 (Android 8.0) |
| 数据来源 | 本阶段用 assets/JSON 模拟网络 API（FakeRemoteDataSource） |
| 禁止 | SharedPreferences、Gson、LiveData、kapt |

---

## 5. 非功能需求

- 首次加载 < 2 秒（数据来自本地 assets）
- 离线可读（首次加载后 Room 缓存）
- 收藏操作即时响应（本地操作无需网络）
- 支持暗色模式
