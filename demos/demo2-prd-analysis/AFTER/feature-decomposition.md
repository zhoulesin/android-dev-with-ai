# 功能拆解

> 由 LLM 解析 PRD 用户故事 → 功能模块 → 开发任务

## 功能树

```
BestPractice App
├── F1 首页文章列表（P0）
│   ├── 1.1 文章卡片组件
│   ├── 1.2 收藏切换交互
│   ├── 1.3 Loading 态
│   ├── 1.4 Error 态（含重试）
│   └── 1.5 首次启动数据加载 + 缓存
├── F2 文章详情（P0）
│   ├── 2.1 详情页布局
│   ├── 2.2 Markdown 正文渲染
│   └── 2.3 导航参数（articleId）
├── F3 收藏管理（P1）
│   ├── 3.1 收藏列表页
│   ├── 3.2 收藏状态持久化
│   └── 3.3 空收藏引导
├── F4 搜索（P1）
│   ├── 4.1 搜索输入框
│   ├── 4.2 实时搜索（200ms 防抖）
│   └── 4.3 搜索空态
└── F5 架构与基础设施（P0）
    ├── 5.1 多模块 Gradle 配置
    ├── 5.2 Hilt 依赖注入
    ├── 5.3 Room Database
    └── 5.4 NavHost 路由
```

## MoSCoW 优先级

| 优先级 | 功能 | 说明 |
|:---:|------|------|
| **Must** | F1 首页列表、F2 详情、F5 基础设施 | MVP 核心 |
| **Should** | F3 收藏管理、F4 搜索 | 完整用户闭环 |
| **Could** | Markdown 富文本渲染 | 目前纯文本展示 |
| **Won't** | 评论、用户系统、推送 | 非 V1 范围 |

## 依赖关系

```
F5 基础设施 ─────┬──→ F1 首页列表 ──→ F2 详情
                 │         │
                 │         └──→ F3 收藏管理
                 │
                 └──→ F4 搜索
```

F5 先做（模块骨架 + Hilt + Room），然后 F1→F2→F3 串行，F4 可并行。
