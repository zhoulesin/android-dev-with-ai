# 时效性标注规范

用于统一本仓库所有章节的时间相关信息标注方式，避免“结论有效期不明”。

## 标注模板（放在每章标题后）

```markdown
> 时效性标注
> - 最后更新：YYYY-MM
> - 统计周期：YYYY-MM ~ YYYY-MM（或 YYYY Qx）
> - 适用说明：模型/工具/价格更新较快，落地前请以官方最新信息复核
```

## 字段定义

- `最后更新`：本章节最后一次内容更新的月份（不是创建日期）
- `统计周期`：章节中主要数据的采样区间；若有多个区间，在文内分段标注
- `适用说明`：固定风险提示，用于强调时效敏感信息需复核

## 适用范围

- 必填章节：`docs/01-12` 所有正文章节
- 可选章节：`README.md`、专题附录、评测报告

## 写法约束

- 统一用 `YYYY-MM` 或 `YYYY Qx`，避免“最近/当前/最新”这类模糊时间词
- 若引用历史数据（如 2025 年），需明确写“历史区间”并避免表达为“当前结论”
- 若章节中包含多个实验，建议在对应小节再补一行“本节统计周期”

## 更新节奏建议

- 常规章节：至少每季度复核一次
- 强时效章节（模型选型、工具对比、Token 成本）：建议每月快速复核，每季度重测
